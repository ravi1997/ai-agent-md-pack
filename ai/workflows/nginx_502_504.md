# Workflow: Nginx 502/504

1) Collect evidence: `checklists/NGINX_502_EVIDENCE.md`
2) Determine failure mode:
- upstream not running → fix service/container
- connect refused → bind mismatch
- connect timeout → network/firewall
- read timeout → long request / timeout mismatch
- bad gateway with import errors → fix env/deps

3) Apply smallest fix (dev/staging)
- adjust bind/socket
- fix env var missing
- increase timeout (only if justified)
- increase workers cautiously

4) Verify
- `nginx -t` + reload
- smoke test
- monitor logs

Artifacts:
- incident report, PR summary if code/config changed
