# Workflow: Feature Delivery (Spec → Tasks → PR)

## Minimal input
One sentence describing the feature.

## Steps
1. Clarify acceptance criteria (max 5 questions; otherwise assume sensible defaults)
2. Break into tasks:
   - backend
   - frontend
   - migrations
   - tests
   - docs
3. Implement incrementally
4. Write tests + run suite
5. Security review for new inputs
6. Produce PR summary + release notes

## Output
- Task list
- Implementation patch
- PR artifact
